MVP A · B · C · D 비교

각 단계는 이전 단계만으로도 운영 가능하도록 나뉜다(중간에 멈춰도 쓸 수 있게). 도구 개발이 워크플로우 설계보다 커지지 않도록, 스크립트는 A에서 3개만 만들고 단계마다 누적한다. 지금 위치: MVP-A 완료.

MVP-A

문서 생성 + readiness

가장 먼저 운영 테스트 가능한 최소 단위. 화면 1개를 끝까지 완주.

✅ 완료 · 코덱스 리뷰 해소
MVP-B

lint-policy 적응

기존 린트가 있는 브라운필드에 충돌 없이 합성.

계획
MVP-C

생성 뷰(generated views)

수동 작성하던 전역 뷰 3종을 전부 생성물로 전환.

계획
MVP-D

Claude Code 통합 + 완성

훅·서브에이전트·플러그인 배포로 1차 방어선 완성.

계획

구성요소 비교 매트릭스

단계별로 새로 추가되는 산출물. new 표시 외에는 이전 단계 것이 그대로 누적된다.

단계별 완료(졸업) 기준

“언제 다음 단계로 넘어가도 되는가”를 결정하는 한 줄 기준.

A 졸업

실프로젝트 화면 1개를 stub→spec→구현→validate 사이클로 완주 ✓

B 졸업

기존 린트가 있는 브라운필드 1곳에 충돌 없이 합성

C 졸업

수동 작성하던 전역 뷰 3종(inventory·route-tree·nav-graph)이 전부 생성물로 전환

D 졸업

Claude Code 플러그인 marketplace 배포 + 훅 5종·semantic-reviewer 동작
