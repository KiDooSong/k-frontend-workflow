# Frontend Workflow Agent Guide

Copy this file to the consumer repository root as `AGENTS.md` or `CLAUDE.md`.
Keep it short: this is the agent starting point, not the full workflow manual.

## Start Here

- Read `docs/frontend-workflow/global/llm-rules.md` for project policy and priority rules.
- When doing a concrete task, read `tools/frontend-workflow/docs/reference/task-artifact-matrix.md`.
- For generated files, read `tools/frontend-workflow/docs/reference/generated-files.md`.
- For command syntax, read `tools/frontend-workflow/COMMANDS.md`.

## Before Editing Code

- For screen work, run or inspect `workflow:state` and `workflow:readiness -- --screen <SCREEN_ID> --json`.
- Read the target ScreenSpec before changing implementation files.
- Only edit readiness `allowed_paths`. Do not edit `forbidden_paths`.
- If using new external input from Figma, planning, API, QA, or meetings, create the input artifact and reconcile it before implementation.

## Common Source Of Truth

- Screen behavior: `docs/frontend-workflow/domains/{domain}/screens/{screen}/screen-spec.md`.
- Navigation: `docs/frontend-workflow/app/navigation-map.md`.
- Project policy: `docs/frontend-workflow/global/llm-rules.md`.
- Shared components: `docs/frontend-workflow/design/component-catalog.md` and `docs/frontend-workflow/global/component-gap-register.md`.
- API contracts: `docs/frontend-workflow/api/api-manifest.md` plus linked contract evidence.

## Hard Boundaries

- Generated files marked `generated/do_not_edit` are regenerated by commands, not hand-edited.
- New shared/common UI components need an approved catalog or a component-gap proposal first.
- API endpoints, DTOs, copy, design values, routes, and selector conventions are not invented.
- Open Decisions, Unknowns, Component Gaps, confirmed status, live policy replacement, and hard CI gates are human-owned transitions unless the user explicitly asks for that transition.

## Before Finishing

- Run the smallest relevant checks, then run `workflow:validate`.
- If route, nav, catalog, policy, lint, or codegen sources changed, regenerate the matching generated view or run the advisory generated-file guard.
- Report which workflow artifacts changed, which generated files were regenerated, and which Open Decisions, Unknowns, or Component Gaps remain open.
