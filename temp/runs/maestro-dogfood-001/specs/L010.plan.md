## Application Overview

L010 is the signup landing: a new user starts signup with a social account (kakao / Google / Apple)
or email, or switches to login. The screen itself takes **no credentials** — it only selects a provider
and routes the choice onward (in this demo, to a placeholder result screen showing the chosen value).

## Plan review (human gate) — what was DROPPED / NOT asserted, and why

- **State Matrix `pending` / `error`: NOT planned.** screen-spec marks them 후속 (OAuth 미연동),
  outside the implemented ceiling (this build is `default`-only). Planning them = asserting unbuilt UI.
- **D-L010-1 (kakao yellow token vs literal): no scenario dropped.** It is a *visual* Open Decision;
  it gates no interaction. Visual fidelity belongs to visual-spec / Figma-diff, **not E2E** (separate axis).
- **U-L010-2 (real destination of "로그인하기"): OPEN.** Plan asserts only the demo placeholder
  ("login"); it does **not** assert a real login screen. Tighten 2.5 after U-L010-2 resolves.
- **Analytics events (`signup_provider_select` / `signup_to_login`): NOT asserted.** No analytics layer
  is wired in this demo; they are evidence-handoff items, not an E2E gate.
